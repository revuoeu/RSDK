# Internationalization


Translations live in the app's static `I18N.set` and are usually passed to the app through `StaticTranslator(I18N.set)`.

## Key format

Use clear, stable keys. For app actions, prefer the fully qualified action name.

```csharp
I18N.set.Translations["en-US"].Entries["SDK.Client.SDKApp.GetSettings"] = "Settings";
I18N.set.Translations["pl-PL"].Entries["SDK.Client.SDKApp.GetSettings"] = "Ustawienia";
```

For UI-only labels, use stable UI keys such as:

```csharp
I18N.set.Translations["en-US"].Entries["UI.Save"] = "Save";
I18N.set.Translations["pl-PL"].Entries["UI.Save"] = "Zapisz";
```

## Runtime access

Use the translator exposed by the current app or control.

```csharp
var actionLabel = Application?.Translator?["SDK.Client.SDKApp.GetSettings"];
var saveLabel = Application?.Translator?["UI.Save"];
```

Some projects expose a shorthand property such as `Application?.T?["UI.Save"]`. Follow the existing project pattern.

## Guidance

- Match action keys to the method name passed to `AddAction`.
- Keep the namespace consistent with `RunAction(...)` usage where possible.
- Do not implement your own fallback logic in controls; `StaticTranslator` is responsible for fallback behavior.
- Rebuild after editing `I18N` files so new keys are picked up.
